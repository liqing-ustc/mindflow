Discussion Notes

1. VLN 现状与核心挑战

  • VLN 解决的是在多房间大场景中的导航问题
  • 现有方法（如 2412-NaVILA）通常输出 waypoint，真实导航仍依赖 SLAM + path planner
  • 相关工作 MTU3D 使用 explicit memory token（latent spatial memory）驱动导航

2. VLA 中的移动能力边界

  • 当前 VLA 的 action space 已包含底盘移动和升降（mobile manipulation）
  • 桌面级移动（操作中的移动）：VLA 擅长，扩充 workspace 范围内的移动
  • 房间级移动（操作间的移动）：VLA 做起来很难——视角有限，遥操采集时人与机器人视角不一致
  • 鱼眼相机的 motivation：扩大视野覆盖范围

3. 大脑-小脑架构（核心共识）

不应区分 VLN 和 VLA，而应区分大脑（high-level planning）和小脑（execution）：

模块角色能力重点部署频率
大脑 (VLM)任务拆解、规划L（语言推理）强云端（千亿/万亿参数）~1 Hz 或 0.5 Hz
小脑 (VLA)执行导航+操作V 和 A 强，L 可以弱端侧（~10B 参数）20-30 Hz
  • 大脑将复杂任务拆解为细粒度 subtask，每个 subtask 可能同时涉及 navigation 和 manipulation
  • 小脑执行类似”条件反射”的动作，不需要太多 reasoning
  • 类比自动驾驶：大脑决定”超车/掉头”，VLA 结合当前观测执行具体轨迹

4. 大脑与小脑之间的桥梁：语言指令

指令粒度的关键讨论：

  • Jack 目前采集数据时标注粗细两种尺度的 L：
    • 粗尺度：如”把衣服从篮子里拿出来，折好”
    • 细尺度：如”甩平衣服 → 折袖子 → 折叠”
  • 核心发现：指令越细、包含的物体越少、动作越简单 → VLA 越容易学习和泛化
    • 例：训练了”拿杯子”的 skill，可迁移到”倒水”任务中的拿杯子步骤
    • 反之，只训”倒水”整体任务，无法迁移回”拿杯子”
  • VLA 的一个难点：判断任务是否结束（本质是 reasoning 问题），目前不够鲁棒、不够泛化

5. 空间理解：视觉锚点 vs. 绝对量化（重要共识）

不应使用绝对数值（“转30度”、“走1米”），而应以视觉特征/物体作为锚点：

  • 大脑指令示例：“向右转90度” → “向右转，直到你看到那个杯子”
  • 类比人类指路：“向前走，看到红灯右转，看到小卖部对面就是”
  • 人类通过 comparison（相对参照）measure 空间，而非绝对尺度
    • 靠步数、身高、臂展等已知量作为参考
    • “厘米/米”是人为定义的系统，非自然产物

好处：

  • 避免大脑需要精确理解”30度是多少”
  • 避免小脑需要追踪已旋转/已移动的量
  • 视觉锚点更容易跨场景泛化
  • 闭环只在小脑内部完成，不受大脑频率瓶颈限制

6. Spatial Memory 的归属

  • 讨论了 spatial memory 应属于大脑还是小脑
  • 共识：memory 更适合放在大脑（VLM）侧
    • 大脑基于全局观测 + 小脑当前局部观测做决策
    • 例：机器人面对沙发，背后有茶几 → 大脑记住茶几位置，发出”转身，把遥控器放到身后茶几上”
    • 小脑不需要全局 memory，只需执行当前指令

7. 数据采集策略

  • 当前方式：定义好的任务，采集员重复执行 → repetitive，多样性不足
  • 新方向（参考 NVIDIA GR00T N1/DreamZero）：给采集员一个场景，让其自由 improvise 任务
    • 速度较慢，但数据更多样
    • 自然带来 navigation 数据（因为要在场景中移动才能发现更多任务）
  • 人类数据（如 Ego-centric video）：
    • 苏一飞同学在处理 experience 数据，含大量 navigation 相关内容
    • 有人头 3D 轨迹，可 project 到地面做动作学习
    • 至少可用于预训练
    • 人的第一视角与机器人视角差异不大（人头 6DOF vs. 机器人 3DOF，影响不大）
  • 五六月计划：拿部分机器 + 已积累的数百/上千个物料，到外部场景 improvise 采集
